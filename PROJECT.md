# PROJECT.md — 화학(과학) 수행평가 준비 패키지 사용 가이드

> 2022 개정 교육과정 **과학과** 수행평가 산출물을 **공식 데이터에 근거해** 만드는 Claude Code 패키지.
> 동료 교사와 공유 시 이 문서부터 읽으세요. (에이전트 동작 규칙은 [CLAUDE.md](CLAUDE.md))

## 무엇을 해주나
한 단원을 교사의 4단계 역할(해석→수업 설계→평가 설계→운영) 순서의 **단원 사이클**로 지원합니다
(상세: [단원-프로토콜.md](단원-프로토콜.md), 평가 구간의 원본은 [평가-프로토콜.md](평가-프로토콜.md)).

```
①해석 ──→ ②수업 설계 ──→ ③평가 설계 ──────→ ④운영 ──────→ 결과 정리 ──────→ 회고 ↻
해석.md    단원설계·형성평가   평가계획서·루브릭·활동지   관찰기록·피드백   채점표·과세특 초안   다음 단원 개선
```

- **① 해석**: 성취기준의 개념 구조 풀기(`교육과정해석` — 원문·내용 체계·수행 동사·연계를 데이터에서 분해)
- **② 수업 설계**: 이해 중심 교육과정(UbD, 백워드 설계) 순서로 짜기 — `단원설계`(단원 핵심 아이디어·핵심 질문 3층위·차시표) + 수행평가 리허설용 `형성평가` 문항지
- **③ 평가 설계(=기존 준비)**: 브레인스토밍(`구상` — 성취기준·탐구활동 × 교과서 4종 요소 × 평가유형 10종으로 후보안 생성) →
  공식 성취기준(교육부 고시 제2022-33호 별책9 원문)에 근거한 평가계획서·루브릭(+기계용 루브릭.json)·활동지
- **④ 운영(=기존 실시)**: 즉석 메모 → 구조화된 관찰기록 CSV (번호·날짜·관찰한 사실·관련 역량) → 관찰 사실 기반 **질문 중심 피드백** 초안(`피드백`)
- **결과 정리**: 관찰기록 × 루브릭 대조 → 근거 있는 채점표 → '사실→역량' 과세특 초안
- **회고**: 채점표 통계 + 수업 운영 회고(차시 계획 대비 실제)로 다음 단원에 고칠 파일 목록 작성

①②는 선택입니다 — 평가만 필요하면 ③부터 시작해도 됩니다(기존 흐름 그대로).

핵심 원칙: **사실은 기억이 아니라 데이터에서 꺼내 쓰고, lint로 검증**합니다.
지어내지 않으며, 모르는 값은 `[확인 필요]`로 비웁니다. ("환각은 구조로 막는다")
채점·과세특은 **관찰기록과 루브릭.json 밖의 근거를 쓰지 않으며**, 모든 점수에 관찰 근거가 남습니다.

## 빠른 시작
Claude Code에서 자연어로 요청하면 알맞은 스킬이 자동 실행됩니다.
- [①해석] "화학 **화학 평형** 단원 **교육과정 해석**해줘" (성취기준 개념 구조 분해)
- [②설계] "이 단원 **단원설계** 만들어줘" (핵심 질문·차시표) → "2차시 **형성평가** 문항 만들어줘"
- [③준비] "화학 **화학 평형** 단원에서 해볼 만한 **수행평가 아이디어** 내줘" (브레인스토밍)
- [③준비] "화학 **화학의 언어** 단원 **평가계획서** 만들어줘" → "**루브릭**도" → "학생용 **활동지**도"
- [④운영] "이 수행 메모를 1반 **관찰기록**으로 정리해줘" (메모 붙여넣기) → "1반 관찰기록으로 번호별 **피드백** 초안 써줘"
- [결과] "1반 관찰기록을 루브릭에 맞춰 **채점표** 만들어줘" → "같은 기록으로 번호별 **과세특 초안** 써줘"
- [회고] "이번 평가 **회고** 정리해줘"

생성물은 `assessments/<과목>-<단원>/`에 저장됩니다.
HTML은 브라우저 인쇄(⌘/Ctrl+P → A4 → PDF 저장) 또는 `bash scripts/html2pdf.sh <파일.html ...>`로 출력합니다(인자 없으면 전체 변환).

## 검증 (중요)
산출물을 만든 뒤 반드시 실행:
```
node scripts/lint.mjs
```
(데스크톱 앱에서는 터미널 없이 **"검증(lint) 돌려서 확인해줘"** 라고 말하면 됩니다 — 에이전트가 위 명령을 대신 실행)
- 성취기준 코드가 해당 과목에 **실재**하는지, **과목↔평가방식**이 맞는지, 과세특 **금지표현·분량**을 점검합니다.
- 프로토콜 산출물도 점검합니다: 루브릭.json 구조·배점 합, 관찰기록 양식·**비식별(번호만)**,
  채점표의 수준·배점이 루브릭에 실재하는지·**근거 없는 점수가 없는지**·합계 검산.
- **✅ 통과**해야 신뢰할 수 있는 초안입니다. (오류가 있으면 종료코드 1)
- `lint.mjs` 자체를 고쳤다면 `node scripts/lint-test.mjs`(자가시험 — 위반을 일부러 주입해 검출되는지 확인)도 통과시켜야 합니다.

## 대상 과목 (2022 개정 과학과 19과목)
- **공통**: 통합과학1·2(상대평가), 과학탐구실험1·2(절대평가·3단계 A·B·C)
- **일반선택**: 물리학·화학·생명과학·지구과학 (상대평가)
- **진로선택**: 역학과 에너지·전자기와 양자·물질과 에너지·화학 반응의 세계·세포와 물질대사·생물의 유전·지구시스템과학·행성우주과학 (상대평가)
- **융합선택**: 과학의 역사와 문화·기후변화와 환경생태·융합과학 탐구 (절대평가·A~E)

## 데이터 갱신
- 교육과정·기재요령이 개정되면 `references/`의 데이터와 각 파일 `_메타`의 **출처·버전**을 교체합니다.
- 성취기준 원문은 `references/_원문/`의 PDF에서 재추출합니다.
- KICE 평가기준·평가 도구는 `python scripts/fetch-eval-criteria.py` / `python scripts/fetch-assessment-tools.py`로 재수집합니다
  (수집 시 별책9 코드와 자동 전수 대조 — 불일치가 뜨면 원문으로 확인).
- 학업성적관리 시행지침은 매 학년도 학교 배포본과 `references/학업성적관리-지침.md`를 대조합니다.

## 사람이 남는 자리 (반드시 읽기)
모든 생성물은 **초안**입니다.
- 채점기준이 우리 반 수준에 맞는지, 과세특 문장이 자연스럽고 공정한지는 **교사가 편집장으로 최종 판단**합니다.
- 데이터 정확성도 담당 과목 파일을 원문과 한 번 대조해 주세요.
- **분량·반영비율·평가시기 등 학교 행정 수치**는 학교 규정으로 확인합니다(`[확인 필요]` 표시).

## 개인정보
학생 실명·개인정보는 산출물에 넣지 않습니다(과세특 초안은 "해당 학생"/변수로 작성).
수업 녹음을 활용할 경우: 사전 공지·동의 하에 녹음하고, **음성 원본·전사 txt는 이 폴더(Drive 동기화) 밖 로컬 전용**에 둡니다 —
패키지에는 비식별 관찰기록 CSV만 들어옵니다. (상세: `관찰기록` 스킬의 '수업 녹취록 가져오기' 모드)

## 저작권 (공유 시 주의)
`references/_교과서/`(원본 PDF)와 `references/수행평가-요소/`(추출 JSON), 그리고 `자료분석/_원본/`(타 교사·기관 연수 자료)은 **로컬 전용**입니다.
동료 교사와 패키지를 공유하거나 공개 저장소에 올릴 때 이 폴더들은 **반드시 제외**합니다.
