# CLAUDE.md — 화학(과학) 수행평가 준비 패키지

> 이 파일은 에이전트(Claude Code) 행동 지침이다. (사용자용 안내는 추후 `PROJECT.md`.)

## 이 패키지가 하는 일
2022 개정 교육과정 **과학과**의 한 단원을 **단원 프로토콜**(①해석→②수업 설계→③평가 설계→④운영→결과 정리→회고, `단원-프로토콜.md`)에 따라
**공식 데이터에 근거해** 반복 수행한다. ③평가 설계~회고 구간의 상세 절차는 **평가 프로토콜**(`평가-프로토콜.md`)이 원본이다.
화학 계열이 1차 대상이나, 구조는 과목 코드로 분기한다.
어떤 산출물을 만들든 먼저 두 프로토콜에서 해당 단계의 근거·산출물·하드룰을 확인한다.

## 핵심 원칙 — 환각은 구조로 막는다
사실에 해당하는 것(성취기준 코드·원문, 평가방식, 학점, 과세특 규정)은 **절대 기억으로 쓰지 않는다.**
반드시 `references/`의 데이터 파일에서 **꺼내 쓴다.**

**근거 라벨 3종** — 산출물의 모든 항목은 출처를 신뢰 등급 순(`원문` ＞ `참고` ＞ `제안`)으로 표기한다:
- `원문` — 고시·공식 문서 그대로 인용(성취기준·핵심아이디어·내용체계·기재요령 등). 세부 출처는 괄호로: `원문(별책9)`.
- `참고` — 출처 분명한 외부 자료(⚠ 고시 원문 아님) — 교과서·지도서·연수 자료. 세부 출처(쪽·저자) 필수: `참고(동아 p.113)`·`참고(임정미)`.
- `제안` — 데이터에 없는 설계 제안(단원 핵심 아이디어·핵심 질문·차시 배분 등). 교사 확정 전 **잠정**임을 명시.
- 상세 정의는 `references/평가유형-카탈로그.md` §4 · `단원-프로토콜.md`. **라벨을 속이지 않는다**(참고·제안을 원문인 척 금지).

## 하드룰 (반드시 지킨다)
1. **과목 먼저 확정** — 어떤 산출물이든 시작 전 `references/교과목록.json`에서 과목 `코드`를 찾아
   **묶음**(공통/일반선택/진로선택/융합선택)과 **평가방식**을 확정한다.
   - 평가방식 3종: `상대평가`(석차등급 1~5 + 성취도 A~E) / `절대평가_5단계`(성취도 A~E) / `절대평가_3단계`(성취도 A·B·C)
   - 평가방식에 맞지 않는 전제로 쓰지 않는다. **예: 융합선택·과학탐구실험에 석차등급 전제 금지.**
2. **성취기준은 원문 인용** — 성취기준 코드는 반드시 `references/성취기준/<과목>.json`에서 찾아
   **코드와 원문을 그대로** 인용한다. 요약·각색·재작성 금지.
3. **없으면 비운다** — 데이터에 없는 코드/내용/수치는 지어내지 말고 `[확인 필요]`로 둔다.
4. **추측 금지** — 학점·평가방식·과세특 규정을 기억으로 채우지 않는다. 데이터에 없으면 `[확인 필요]`.
5. **검증 필수** — 산출물을 만들거나 고칠 때마다 `node scripts/lint.mjs`를 실행해 **통과**시킨다.
   `lint.mjs` 자체를 수정했다면 `node scripts/lint-test.mjs`(자가 시험 — 위반 주입 후 검출 확인)도 통과시킨다.
6. **비식별** — 학생은 어떤 파일에서든 **번호로만** 다룬다. 실명·학번·생년월일 금지 (lint가 번호 열 검사).
   입력 메모에 실명이 있으면 번호로 치환하고 치환 사실을 교사에게 보고한다.
   수업 **녹음 음성·전사 txt는 실명 포함 가능** → 이 폴더(Drive 동기화) 밖 로컬 전용에 두고, 패키지에는 비식별 CSV만 들인다.
   녹취록의 화자 미상 발화를 특정 학생에게 추정 배정하지 않는다 (번호 확정은 교사).
7. **채점은 근거 파일 밖으로 나가지 않는다** — 채점표·번호별 과세특의 근거는
   `수행기록/관찰기록-<반>.csv` + `루브릭.json` **뿐**이다. 기록에 없으면 지어내지 않고 `[확인 필요]`.
   모든 점수에 근거(관찰 날짜·사실 요약)를 남긴다 (lint가 수준·배점·근거 검사).

## 데이터 위치 (진실의 원천)
- `references/교과목록.json` — 과목 코드·묶음·평가방식·학점 분기표 (2022 개정 과학과 19과목)
- `references/성취기준/<과목>.json` — 과목별 성취기준 코드+원문, 영역, 탐구활동, 성취기준 해설, 적용 시 고려 사항, **핵심아이디어**(별책9 내용 체계 원문, 과목 단위 배열), **내용체계**(별책9 원문 — 지식·이해는 영역별, 과정·기능·가치·태도는 과목 단위)
- `references/평가기준/<과목>.json` — 성취기준별 **평가기준(성취수준 A~E / 상·중·하)** 원문. 출처: KICE 학생평가지원포털(stas.moe.go.kr). **루브릭 수준 기술의 `원문(KICE 평가기준)` 근거.** 코드는 성취기준(별책9)과 전수 일치(lint 2c 검증). 수집: `python scripts/fetch-eval-criteria.py`
- `references/_원문/` — 교육부 고시 제2022-33호 [별책 9] 원문 PDF·텍스트 + **2026 학교생활기록부 기재요령(고등학교, 교육부)** — 과세특 규정 근거 (검증·갱신 근거)
- `references/학업성적관리-지침.md` — 경남 학업성적관리 시행지침 수행평가 관련 **원문 발췌**(횟수·40% 규칙·과제형 금지·보관 등 — 평가계획서 하드룰). 원문 HWP는 `_원문/`
- `references/수행평가-도구-KICE.json` — KICE 학생평가지원포털 고교 과학 평가 도구 카탈로그 269건(⚠ 대부분 2015 개정 — 아이디어 차용용 `참고`). 수집: `python scripts/fetch-assessment-tools.py`
- `references/관찰기록-양식.md` — 실시 단계 관찰기록 CSV 규격(4열·비식별)
- `references/평가유형-카탈로그.md` — 수행평가 유형 10종 + 교과서 요소 분류 어휘 8종 + 개념적 렌즈 10종 + 수행지시어 사전 + 핵심 질문 3층위 (구상·루브릭·단원설계의 근거. §4~§6은 고시 원문 아님)
- `references/수행평가-요소/<과목>-<출판사>.json` — 교과서·지도서에서 추출한 수행평가 요소 (활동명·유형·영역·쪽)
- `references/_교과서/` — 교과서·지도서 원본 PDF. **위 두 항목과 함께 로컬 전용 — 공유 저장소에 절대 포함 금지 (저작권)**
- `references/A4-인쇄-CSS.md` — HTML 산출물 공통 인쇄 스타일

## 사람이 남는 자리
사실 정확성은 데이터+lint로 해결한다. 그러나 **교육적 판단**(이 채점기준이 우리 반 수준에 맞나,
이 과세특 문장이 자연스럽고 공정한가)은 교사가 **편집장**으로 남는다. 모든 생성물은 **초안**이며 교사 검토를 전제한다.

## 작업 규약
- 변수명·함수명은 영어. 데이터/산출물의 키와 내용은 한글(교사가 직접 검증).
- 응답·주석·커밋 메시지는 한국어.
