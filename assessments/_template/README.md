# _template/ — 새 과제 시작용 뼈대

새 수행평가 과제를 시작할 때 이 폴더를 복사해 `assessments/<과목>-<과제명>/`으로 이름을 바꾼다.
(예: `assessments/화학-화학평형/`) 이 `_template/` 자체는 lint 검사에서 제외된다.

## 단원 사이클 단계별 구성 (상세: `단원-프로토콜.md` · `평가-프로토콜.md`)

```
<과목>-<과제명>/
├── meta.json                      과목·성취기준·반영비율 — 과목코드·묶음·평가방식은 교과목록.json과 일치(lint 검증)
├── 해석.md                        [①해석] `교육과정해석` 스킬이 생성 — 성취기준 개념 구조·동사 분석·연계
├── 단원설계.md                    [②설계] `단원설계` 스킬이 생성 — 핵심 질문·차시표(형성=리허설 배치)
├── 형성평가-<차시>차시.html        [②설계] `형성평가` 스킬이 생성 — 수행평가 리허설용 문항지
├── 구상.md                        [③준비] 브레인스토밍 후보안 (선택 — `구상` 스킬)
├── 평가계획서.html                 [③준비] `평가계획서` 스킬이 생성
├── 루브릭.html · 루브릭.json       [③준비] `루브릭` 스킬이 생성 — 사람용·기계용 쌍(둘 다 필수)
├── 활동지.html                    [③준비] `활동지` 스킬이 생성
├── 수행기록/                      [④운영] 관찰기록-<반>.csv (안내: 수행기록/README.md)
├── 산출물/                        [④운영·결과] 피드백·채점표·과세특 초안 (안내: 산출물/README.md)
└── 회고.md                        [회고] `회고` 스킬이 채움 (+수업 운영 회고)
```

- 생성형 산출물(해석·단원설계·형성평가·평가계획서·루브릭·활동지 등)은 템플릿에 없다 — 해당 스킬이 데이터에 근거해 생성한다.
- ①②는 선택이다 — 평가만 만들 때는 [③준비]부터 시작해도 된다(기존 평가 프로토콜 그대로).
- `meta.json`의 대괄호 자리는 **교과목록.json에서 확정**해 채운다. 모르는 값은 `[확인 필요]`.
- 산출물을 만들거나 고칠 때마다 `node scripts/lint.mjs` 통과.
