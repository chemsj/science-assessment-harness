# 교육과정해석 스킬 (단원 사이클 ①단계 — 교육과정 해석)

## 언제 쓰나
새 단원을 시작할 때 **가장 먼저**. 구상·평가계획서를 쓰기 **전에**, 그 영역의 성취기준이
어떤 **개념 구조**로 짜여 있는지(무엇을 알고·무슨 사고를 시키며·어디와 잇닿는지)를 데이터에서 풀어낸다.
교사의 4단계 역할 중 **① 교육과정 해석(교육과정 재구성자)** 에 해당한다. (`단원-프로토콜.md` 단계 ①)
> ⚠ 4단계 역할·개념기반 교육과정과 수업(CBCI; Erickson, Lanning & French, 2017) 해석은 방법론 차용이며 **고시 원문이 아니다** (근거: `자료분석/` 임정미 연수 자료). 원문 인용 항목과는 라벨로 구분한다.

## 입력 감지
- **과목** (필수): `references/교과목록.json`의 `코드` (예: `화학`, `물질과에너지`).
- **단원/영역** (필수): 해당 과목 성취기준 파일의 `영역명` 또는 영역번호. 없으면 사용자에게 확인.
- **학년·학기** (선택): 모르면 `[확인 필요]`.

## 근거 데이터 (꺼내 쓰는 순서)
1. `references/교과목록.json` — 과목 `코드` → **묶음·평가방식**(상대/절대_5단계/절대_3단계) 확정.
2. `references/성취기준/<과목>.json` — 대상 영역의 **성취기준 코드+원문**, **탐구활동**, **성취기준해설**,
   **적용시고려사항**, 과목 단위 **`핵심아이디어`**, 그리고 **`내용체계`**(있으면):
   `{ "지식·이해": { "<영역명>": [내용 요소…] }, "과정·기능": […], "가치·태도": […] }`.
3. `references/평가유형-카탈로그.md` — **§5 수행지시어 사전**(동사 위계) · **§4 개념적 렌즈**(방향 참고). ⚠ §4·§5는 고시 원문이 아닌 패키지 내부 규약이다.

## 절차 (원문 인용 ↔ 교사 판단 보조를 라벨로 구분)
1. **과목 확정** — 교과목록에서 코드를 찾아 **묶음·평가방식**을 확정한다. 목록에 없으면 중단·확인(추측 금지).
   평가방식은 이후 루브릭·채점 설계의 전제이므로 해석.md 머리에 명시한다(상대평가면 변별 여지도 관점에 둔다).
2. **성취기준 원문 표** — 대상 영역의 성취기준을 **코드+원문 그대로**(요약·각색 금지) 표로 옮긴다 → 라벨 `원문(별책9)`.
3. **내용체계 분해** → 라벨 `원문(별책9)` + 매핑은 라벨 `제안`(교사 판단 보조):
   - `내용체계`에서 **해당 영역의 지식·이해 내용 요소**(원문)와 과목의 **과정·기능·가치·태도**(원문)를 그대로 꺼낸다.
   - 성취기준마다 어떤 지식·이해 요소·과정·기능과 맞닿는지 **매핑 표**를 만든다 — 이 매핑은 교사 판단 보조이므로 `제안` 라벨.
   - **`내용체계` 필드가 데이터에 없으면** 해당 내용 요소를 지어내지 말고 `[확인 필요]`로 두고, 데이터 보강을 안내한다.
4. **성취기준 동사 분석** — 각 성취기준 원문에서 **수행 동사**("설명할 수 있다"·"예측할 수 있다"·"조사·발표"·"판단할 수 있다" 등)를 뽑아
   카탈로그 §5 **수행지시어 위계**(설명·논증 ＞ 서술·적용 ＞ 해석·분류 ＞ 요약 ＞ 나열)에 대응시킨다 → 요구되는 **사고 수준** 표시.
   동사는 원문에서 추출(`원문(별책9)`), 위계 대응은 판단 보조(`제안`).
5. **연계 지도** — `성취기준해설`·`적용시고려사항` 원문을 인용해 **선수 학습·타 과목 연계·유의점(안전 포함)** 을 정리 → 라벨 `원문(별책9)`(그대로 인용).
   해당 영역에 이 필드가 없으면 지어내지 않고 `[확인 필요]`로 비운다.
6. **교육과정 핵심아이디어** — 과목 `핵심아이디어`(별책9) 중 **대상 영역과 맞닿는 문장**을 표시 → 라벨 `원문(별책9)`.
   (여기서 만든 것은 교육과정 핵심 아이디어일 뿐, **단원 핵심 아이디어(제안)는 구상 단계**에서 렌즈로 만든다 — 섞지 않는다.)
7. **교사 확정란 + 다음 단계 안내** — 해석 동의·수정 메모란을 두고, 다음 스텝(`구상` 또는 `단원설계`)으로 안내.
8. **meta.json 처리 + lint**:
   - 폴더 `assessments/<과목>-<단원>/`에 `meta.json`이 **없으면** 초안 생성(미정 값은 `[확인 필요]`, `성취기준코드`는 대상 영역 코드로 채움, `산출물`에 `해석.md`).
   - **있으면** `산출물` 목록에 `해석.md`를 추가한다.
   - `해석.md` 저장 후 `node scripts/lint.mjs` **통과**(meta 산출물 목록과 실제 파일 일치, 성취기준코드 실재 검사).

## 해석.md 구조
1. **머리** — 과목명 · 묶음 · **평가방식** · 영역명 / 학년·학기(모르면 `[확인 필요]`) / 라벨 범례(`원문`=고시 원문 인용, `제안`=교사 판단 보조).
2. **[원문] 성취기준 원문 표** — `코드 | 성취기준 원문(그대로)`.
3. **[원문] 내용체계 분해** — 지식·이해(영역별 내용 요소) / 과정·기능 / 가치·태도 원문 목록. + **[제안] 성취기준↔내용요소·과정기능 매핑 표**.
4. **[원문→제안] 동사 분석 표** — `코드 | 수행 동사(원문) | §5 위계 | 요구 사고 수준`.
5. **[원문] 연계 지도** — 성취기준해설·적용시고려사항 인용(선수 학습·타 과목·유의점).
6. **[원문] 교육과정 핵심아이디어(영역 관련)** — 맞닿는 문장 표시.
7. **교사 확정란** — 동의/수정 메모 + 다음 단계(`구상`/`단원설계`) 안내.
8. 하단: 출처(교육부 고시 제2022-33호 [별책 9]) + "초안이며 교사 확정 필요".

## 핵심
- **원문은 원문으로, 판단은 판단으로** — 인용 항목은 `원문`, 매핑·위계 대응·시사점은 `제안`. 라벨을 속이지 않는다.
- **없으면 비운다** — `내용체계` 등 데이터에 없는 내용 요소를 지어내지 않고 `[확인 필요]`.
- **두 핵심 아이디어를 섞지 않는다** — 여기서는 교육과정 핵심아이디어(별책9)만 다룬다. 단원 핵심 아이디어(제안)는 `구상`에서.
- 해석은 **초안**이다. 개념 구조의 확정·재구성은 교사(편집장)의 몫이며, 그 결과가 `구상`·`단원설계`의 입력이 된다.
