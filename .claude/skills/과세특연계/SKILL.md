# 과세특연계 스킬

## 언제 쓰나
수행평가·루브릭 결과를 학교생활기록부 **과목별 세부능력 및 특기사항(과세특)** 초안으로 옮길 때.

## 입력 감지
- **과목** (필수): `references/교과목록.json`의 코드.
- **단원/성취기준** (필수).
- **관찰기록·채점표** (권장): `수행기록/관찰기록-<반>.csv` + `산출물/채점표-<반>.csv`가 있으면
  **번호별 초안**을 만든다 (평가 프로토콜 단계 3). 없으면 수준별 예시 상/중/하만 생성.

## 절차 (CLAUDE.md + 과세특-기재요령.md 준수)
1. `교과목록.json` → 과목 확정.
2. `성취기준/<과목>.json`에서 대상 성취기준을 문장 근거로 인용.
3. `references/과세특-기재요령.md`의 **금지표현·분량·작성 원칙**을 적용한다.
4. 관찰기록·채점표가 있으면 **번호별 '사실 → 역량' 변환**으로 작성한다:
   - 관찰기록의 사실 문장에서 출발해 역량 서술로 잇는다.
     (예: "종말점 근거를 모둠에 설명함" → "실험 데이터를 해석하고 동료와 공유하는 의사소통 역량을 보임")
   - **관찰기록에 없는 사실을 지어내지 않는다.** 기록이 빈약한 학생은 문장을 짧게 두고 `[확인 필요]` 표시.
   - 채점표의 수준은 문장 강도의 참고로만 쓴다 (점수·등급 언급 금지).
5. 성취기준·루브릭 수준에 근거해 **관찰 사실 문장**을 작성한다:
   - **금지표현 금지**(수상·인증시험·모의고사·논문·도서 출간·특허·해외활동·부모정보·대학명 등).
   - 분량은 `references/과세특-기재요령.md`의 상한을 따른다 — **과목별 500자**(한글 기준.
     NEIS 산정: 한글 3Byte·영문/숫자/공백 1Byte = 상한 1,500Byte. 원문(2026 기재요령 [참고자료 8])). 초과하면 lint가 경고한다.
   - ⚠ **공통과목(통합과학·과학탐구실험 등)은 1·2 합산 500자**(2022 개정 적용 학년) —
     통합과학1·2 각각 500자가 아니라 합산 분량 안에서 배분. 학교 지침이 우선.
   - **학생 실명·개인정보 금지** → "해당 학생" 또는 `[학생]` 변수.
   - 추측·과장 금지. "~함", "~을 보임" 등 **관찰 서술**.
6. 저장 위치: 번호별 초안은 `산출물/과세특-초안-<반>.md`(번호를 `## <번호>번` 섹션으로),
   수준별 예시는 `과세특-초안.md`. `meta.json`의 `산출물`에 추가.
7. `node scripts/lint.mjs` 통과(금지표현·분량 검사 — 산출물/ 하위 포함).

## 과세특-초안.md 구조
- 머리: 과목·단원·성취기준 코드 / 규정 출처(`references/과세특-기재요령.md`)
- 수준별(`## 상 수준` 등) 또는 학생별 과세특 문장 — 각 **한글 500자(1,500Byte) 이내**, `##` 섹션으로 구분(lint가 섹션별 NEIS 바이트 검사)
- 하단: "초안·교사 검토 필요", 실명·개인정보 미포함 안내

## 핵심
- **사실 정확성**(성취기준·규정)은 데이터로, **문장의 자연스러움·공정성**은 교사가 편집한다.
- 금지표현 절대 금지. 분량 준수. 실명 금지.
