# 활동지 생성 스킬

## 언제 쓰나
학생이 직접 수행하는 탐구·실험 **활동지(워크시트)**를 만들 때.

## 입력 감지
- **과목** (필수): `references/교과목록.json`의 코드.
- **단원/성취기준** (필수).
- **활동 형태**(탐구/실험/조사 등, 선택).

## 절차 (CLAUDE.md 하드룰 준수)
1. `교과목록.json`에서 과목 확정.
2. `성취기준/<과목>.json`에서 대상 영역의 **성취기준**과 **`탐구활동`** 항목을 원문 그대로 가져온다.
3. 탐구활동을 학생이 따라 할 수 있는 단계로 구조화: **목표 → 준비물 → 과정 → 결과 기록 → 정리·토의**.
4. **안전 유의사항**이 `적용시고려사항`/`성취기준해설`에 있으면 반영(예: 보호 장구, 폐액 처리).
5. 모르는 값(차시·준비물 세부 등)은 `[확인 필요]`.
6. `assessments/<과목>-<단원>/활동지.html` 생성. `meta.json`의 `산출물` 배열에 `"활동지.html"` 추가.
7. `node scripts/lint.mjs` 통과.

## 활동지.html 구조 (A4)
- 머리: 과목·단원·성취기준 / 학년·반·번호·이름 기입란
  - ⚠ 성취기준 **코드를 병기하면 그 원문 전문도 같은 파일 안에** 실어야 한다(하드룰 2 — lint가 검사).
    코드만 배지로 넣고 원문을 요약하면 lint 오류가 난다.
- **탐구 목표**: 성취기준 기반
- **활동별 구성**: 과정 안내 + 학생 **기록란**(표·빈칸 충분히)
- **정리·토의** 질문
- 하단 `.foot`: 출처 + 안전·유의 + 교사 안내

## 핵심
- 성취기준·탐구활동 **원문 근거**. 지어내지 않는다.
- 학생 기록 공간을 넉넉히. 안전 수칙은 데이터에 있으면 반드시 포함.
- `references/A4-인쇄-CSS.md` 스타일. 미확정은 `.todo`(`[확인 필요]`).
