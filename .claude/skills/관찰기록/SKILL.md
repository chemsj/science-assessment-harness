# 관찰기록 스킬 (실시 단계)

## 언제 쓰나
수행평가·수업 중 남긴 즉석 메모, 또는 수업 녹음·녹취록을
**결과 단계에서 근거로 쓸 수 있는 표**로 바꿀 때. (평가 프로토콜 단계 2 — `평가-프로토콜.md`)

## 입력 감지
- **과제 폴더** (필수): `assessments/<과목>-<과제명>/`.
- **반** (필수): 파일명에 들어감.
- **원천 자료** (필수, 셋 중 하나):
  - A. **즉석 메모** — 대화에 붙여넣거나 파일 경로로 (수첩·붙임쪽지·타이핑 원문).
  - B. **수업 녹취록(전사 텍스트)** — 출처 무관: `scripts/transcribe.py` 산출물, 노션 AI 회의노트, 클로바노트 등.
  - C. **수업 녹음 음성 파일** (mp3·m4a·wav 등) — 먼저 전사해 B로 만든다:
    `uv run scripts/transcribe.py <음성파일>` (Mac/Windows 겸용. 첫 실행은 모델 다운로드,
    CPU 기기는 녹음 길이의 절반~한 배 소요 — 배치 운영 권장. 상세는 스크립트 머리말).

## 절차 A — 즉석 메모 (`references/관찰기록-양식.md` 준수)
1. 양식 문서에서 **열 정의·규칙**을 확인한다: `번호,날짜,관찰한 사실,관련 역량` 4열 고정.
2. 메모를 **한 관찰 = 한 행**으로 분해한다.
3. 변환 규칙:
   - 학생 식별자는 **번호(숫자)만**. 메모에 이름이 있으면 **번호로 치환**하고, 번호를 모르면 `[확인 필요]`로 두고 교사에게 알린다. 실명은 결과 파일에 절대 남기지 않는다.
   - '관찰한 사실'은 **사실 문장**으로: 해석·평가어("우수함","성실함")가 메모에 있으면 근거 행동으로 풀어 쓰되, **메모에 없는 행동을 지어내지 않는다.** 풀 수 없으면 그대로 두고 `[확인 필요]` 표시.
   - '관련 역량'은 짧은 명사구(`·` 구분). 메모에서 유추 가능한 범위까지만.
   - 쉼표가 든 문장은 큰따옴표로 감싼다.
4. `수행기록/관찰기록-<반>.csv`로 저장 (기존 파일이 있으면 **행 추가**, 중복 행 제거).
5. `node scripts/lint.mjs` 통과 (헤더·번호 숫자 검사).

## 절차 B/C — 수업 녹취록(전사) 가져오기
1. C(음성)면 `transcribe.py`로 전사부터. 전사 txt는 음성 파일 옆(로컬 전용 폴더)에 생긴다.
2. 녹취록에서 **명백한 화학 용어 오인식만** 문맥으로 교정한다(예: "물 농도"→"몰 농도"). 의미를 바꾸는 수정 금지.
3. **관찰 후보 목록**을 만들어 교사에게 제시한다: 질문·오개념 발화·발표·실험 수행 장면을
   `[시각] 장면 요약` 형태로 추린다. 수업 전체 요약이 아니라 **학생 행동·발화 후보**만.
   음성 태깅(아래 규약)이 있는 후보는 번호를 미리 채워 제시한다.
4. **번호 확정은 교사만 한다.** 녹취록에는 화자 정보가 없다 —
   태깅 없는 발화를 특정 학생에게 **추정 배정하는 것 금지** (하드룰: 지어내지 않는다).
   - 음성 태깅으로 번호가 명시된 후보도 **교사 확인 후** CSV에 반영한다 (전사 오인식 가능).
   - 교사가 번호를 확정한 후보만 CSV 행으로 변환한다. 번호 미상 후보는 CSV에 넣지 않고 목록에 남긴다 (lint가 번호 열 숫자 검사).
   - 녹취록에 실명 호명이 있으면 번호로 치환하고 치환 사실을 보고한다.
5. 날짜는 녹음 파일명·교사 확인으로 (권장 파일명: `YYYY-MM-DD_반_차시.m4a`).
6. 이후 저장·lint는 절차 A의 4~5와 동일. '관찰한 사실'은 사실 문장 규칙 그대로 (예: "수소·산소 부피비가 2:1이 되는 이유를 질문함").

### 음성 태깅 규약 (권장 — 녹취록의 번호 매칭을 살리는 교사 발화 습관)
- **호명은 번호로**: "7번, 발표해 볼까" — 이어지는 발화를 7번 후보로 연결할 수 있고, 실명 호명보다 비식별 원칙에도 맞다.
- **관찰 구술은 마커 "기록"으로 시작**: "기록, 7번, 단위 소거 과정을 스스로 점검함."
  수업 직후·쉬는 시간에 녹음이 켜진 상태로 구술하면 기억이 사라지기 전에 남는다 (손 안 대는 즉석 메모).
- ⚠ **혼동 주의**: 수업 담화의 "N번"은 대개 **문제 번호**다("3번 문제 풀어보세요").
  마커 없는 "N번"은 자동 매칭하지 않는다 — `기록, N번` 또는 `N번 학생`처럼 학생임이 분명한 형태만 번호 후보로 삼는다.
- **공개 호명 주의**: 학급 전체가 듣는 상황에서 평가성 발언과 번호를 묶지 않는다 (예: "기록, 5번, 계산 틀림" ✗ 공개 구술).
  민감한 관찰은 학생이 없는 구간(수업 직후)에 구술한다. 구술도 평가어 없이 **사실 문장**으로 (양식.md 규칙 동일).

## 개인정보 (녹음 관련 추가 규칙)
- 음성 원본·전사 txt는 **Google Drive 동기화 폴더(이 패키지) 밖** 로컬 전용 폴더에 둔다
  (예: Mac `~/Documents/수업녹음/`, Windows `문서\수업녹음\`). 패키지에 들어오는 것은 **비식별 관찰기록 CSV뿐.**
- 녹음은 학생 **사전 공지·동의** 하에. 동의서 문구·보관 기간·폐기 규정은 학교 지침 확인 `[확인 필요]`.
- 전사 후 음성 원본 폐기 시점은 교사 판단 (조기 폐기 권장).

## 핵심
- 기록의 **원천은 교사의 메모·확인**이다. 스킬은 구조화만 한다 — 내용을 보태지 않는다.
- 녹취록은 **기억 회수 보조**다: 교사가 번호를 확정하기 전에는 채점 근거가 되지 않는다.
- 실명 발견 시 치환 사실을 교사에게 보고한다 (원본 메모·녹음 파기는 교사 판단).
