# 평가계획서 생성 스킬

## 언제 쓰나
교사가 특정 **과목·단원**의 수행평가 **평가계획서**를 만들 때. 평가요소, 반영비율, 평가시기,
성취기준 매핑, 평가방식(상대/절대)을 한 장으로 정리한다. (세부 채점기준표는 `루브릭` 스킬이 담당.)

## 입력 감지
요청에서 다음을 파악한다. 불명확하면 진행 전 묻는다.
- **과목** (필수): `references/교과목록.json`의 `코드` 중 하나 (예: `화학`, `물질과에너지`, `통합과학1`).
- **단원/영역** (권장): 성취기준 파일의 `영역명` 또는 영역번호. 없으면 전 영역 또는 사용자 확인.
- **학년·학기 / 반영비율 / 평가시기** (선택): 학교 사정. 모르면 `[확인 필요]`.

## 절차 (CLAUDE.md 하드룰 준수)
1. `references/교과목록.json`에서 과목 `코드`를 찾아 **묶음·평가방식·성취기준파일**을 확정한다.
   목록에 없으면 중단하고 사용자에게 확인한다. (추측 금지)
2. `references/성취기준/<과목>.json`에서 대상 영역의 **성취기준 코드+원문**을 그대로 가져온다.
3. **평가요소**를 성취기준에 매핑한다. 각 평가요소는 1개 이상의 성취기준 코드에 연결한다.
   - (선택) 수행과제가 **실제 맥락**을 갖도록 **GRASPS**(이해 중심 교육과정(UbD, 백워드 설계)의 수행과제 맥락 틀) 블록을 넣는다 — 목표(Goal)·역할(Role)·청중(Audience)·상황(Situation)·결과물(Product)·준거(Standards). 예: "과학 잡지 기자가 되어, 구독자(청소년)에게, 데이터로 결론을 담은 분석 보고서를 쓴다." 준거는 평가요소와 일치시킨다.
   - GRASPS는 설계 프레임(UbD)이며 **고시 원문이 아니다** — 넣을지는 교사 선택. `구상` 단계나 `단원설계.md`의 GRASPS 블록에서 잡았으면 그대로 옮긴다.
4. **평가방식 분기**:
   - `상대평가` → 석차등급(1~5) + 성취도(A~E). 변별 가능한 배점으로 설계.
   - `절대평가_5단계` → 성취도(A~E)만. 성취수준 기준 명시. (융합선택)
   - `절대평가_3단계` → 성취도(A·B·C)만. (과학탐구실험)
4b. **지침 하드룰 확인** — `references/학업성적관리-지침.md`(경남, 원문 발췌)의 규정과 어긋나지 않게 설계한다:
   지필 2회↑/수행 1회↑(예외 3종 — 수행 60%↑면 지필 1회 가능·수행만 평가 시 영역 달리 2회↑), 수행평가는 환산총점의 40%↑,
   과제형 수행평가 금지(수업 중 실시), 모둠평가는 개인별 역할 평가, 배점×반영비율에서 무한소수 방지.
   ⚠ 수치는 시도마다 다르다(예: 인천은 서·논술형 20%↑) — **경남 지침·학교 규정이 우선.**
5. 학교별 값(반영비율·시기·학년 등)을 모르면 `[확인 필요]`로 둔다. **지어내지 않는다.**
5b. (권장 — `자료분석/시도교육청-평가자료-분석.md`의 인천 양식 차용) 다음 절을 계획서에 포함할 수 있다:
   - **성취기준 커버리지**: 과목(단원)의 모든 성취기준이 지필·수행 어디서 평가되는지, 평가하지 않는 코드는 대체 확인 방법.
   - **최소 성취수준 보장지도**(1학년 전 과목 의무 — 경남 지침 제10조⑩).
   - **부록: 성취기준별 평가기준** — `references/평가기준/<과목>.json`(KICE)에서 자동 생성 가능.
6. 산출물 폴더 `assessments/<과목>-<단원>/`를 만들고 아래 두 파일을 생성한다.
   - `meta.json` (스키마 아래)
   - `평가계획서.html` (구조 아래, `references/A4-인쇄-CSS.md` 스타일 사용)
7. `node scripts/lint.mjs`를 실행해 **통과**시킨다. 실패하면 고쳐서 다시 통과시킨다.

## meta.json 스키마
```json
{
  "과목코드": "화학",
  "묶음": "일반선택",
  "평가방식": "상대평가",
  "학년": "[확인 필요]",
  "학기": "[확인 필요]",
  "단원": "화학의 언어",
  "성취기준코드": ["[12화학01-01]", "[12화학01-02]"],
  "반영비율": "[확인 필요]",
  "평가시기": "[확인 필요]",
  "산출물": ["평가계획서.html"]
}
```
- `과목코드·묶음·평가방식`은 교과목록.json과 **반드시 일치**(lint가 검사).
- `성취기준코드`는 해당 과목 성취기준 파일에 **실재**해야 한다(lint가 검사).

## 평가계획서.html 구조 (A4 세로)
1. **머리**: 과목명 · 묶음 · 평가방식 배지 · 학년/학기
2. **표1 평가 개요**: 평가명 / 유형(수행평가) / 반영비율 / 평가시기
2-1. **(선택) 수행과제 맥락(GRASPS)**: 목표·역할·청중·상황·결과물·준거 6줄 표
3. **표2 평가요소 ↔ 성취기준**: `평가요소 | 성취기준 코드 | 성취기준 원문(그대로) | 배점`
4. **표3 평가방식·산출**: 평가방식에 따라 — 상대평가는 석차등급 산출 명시, 절대평가는 성취도 단계 명시
5. **성취수준 개요**: 상/중/하(또는 A~E) 도달 기준 한두 줄 (상세 채점표는 루브릭 스킬)
6. **하단 `.foot`**: 출처(교육부 고시 제2022-33호 [별책 9]) + "초안이며 교사 검토 필요" + 교사 확인란

## 스타일·태도 핵심
- A4 인쇄 가정. `references/A4-인쇄-CSS.md`의 공통 CSS를 `<style>`에 포함.
- 성취기준 **원문은 수정 없이** 표에 넣는다. 코드는 `.code`.
- 모든 미확정 값은 `.todo`(`[확인 필요]`)로 강조해 교사가 채우게 한다.
- 생성 후 반드시 lint 통과. 결과물은 **초안**임을 명시한다.
