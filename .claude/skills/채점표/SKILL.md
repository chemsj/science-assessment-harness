# 채점표 스킬 (결과 정리 단계)

## 언제 쓰나
실시 단계의 관찰기록을 루브릭에 연결해 **근거 있는 채점 초안**을 만들 때.
(평가 프로토콜 단계 3 — `평가-프로토콜.md`. 관찰 기록 → 루브릭 채점 → 점수)

## 입력 감지
- **과제 폴더** (필수): `assessments/<과목>-<과제명>/`.
- **반** (필수): `수행기록/관찰기록-<반>.csv`가 있어야 한다. 없으면 중단하고 `관찰기록` 스킬 안내.
- **루브릭.json** (필수): 없으면 중단하고 `루브릭` 스킬로 먼저 생성 안내.

## 절차 (근거 밖 채점 금지)
1. `루브릭.json`을 읽어 **평가요소·약칭·수준명·배점**을 확정한다. **이 파일 밖의 수준·배점을 쓰지 않는다.**
2. `수행기록/관찰기록-<반>.csv`를 읽어 **번호별로 관찰 사실을 모은다.**
3. 번호별·평가요소별 판정:
   - 관찰 사실이 어느 수준 기술에 부합하는지 **루브릭.json의 기술 문구와 대조**해 수준을 정한다.
   - 배점은 해당 수준의 배점을 **그대로** 쓴다 (임의 점수 금지).
   - **근거 열에 판단에 사용한 관찰(날짜+사실 요약)을 기록한다.** 근거 없는 점수 금지.
   - 해당 요소에 대한 관찰이 없거나 판정하기에 부족하면 수준·점수·근거 모두 `[확인 필요]`. **지어내지 않는다.**
4. `산출물/채점표-<반>.csv` 저장. 열 형식 (루브릭.json의 약칭 사용):
   ```csv
   번호,<약칭1>_수준,<약칭1>_점수,<약칭1>_근거,<약칭2>_수준,...,합계,비고
   ```
   - `합계`: 모든 요소가 채점된 경우만 산출. `[확인 필요]`가 하나라도 있으면 합계도 `[확인 필요]`.
   - `비고`: 보완 관찰이 필요한 요소 등 교사 확인 사항.
5. 요약 보고: 채점 완료 인원 / `[확인 필요]` 인원·요소 / 수준 분포.
6. `node scripts/lint.mjs` 통과 (수준·배점이 루브릭.json에 실재하는지, 번호가 관찰기록에 있는지 검사).

## 핵심
- **근거는 두 파일뿐**: 관찰기록 CSV + 루브릭.json. 그 밖의 기억·추측으로 채점하지 않는다.
- 채점표는 **초안**이다. 모든 점수와 `[확인 필요]` 해소는 교사가 확정한다.
- 관찰기록이 부실할수록 `[확인 필요]`가 늘어나는 것이 **정상 동작**이다 (보완 관찰 신호).
