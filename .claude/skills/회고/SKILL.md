# 회고 스킬 (회고 단계 — 순환을 닫는다)

## 언제 쓰나
한 과제의 채점·과세특까지 끝난 뒤. 이번 평가에서 흔들린 것을 기록해
**다음 준비 단계의 입력**으로 만들 때. (평가 프로토콜 단계 4 — `평가-프로토콜.md`)
단원 사이클(`단원-프로토콜.md`)로 진행한 과제라면 교육과정 해석·단원 설계·수업 운영까지 회고 대상이다.

## 입력 감지
- **과제 폴더** (필수): `assessments/<과목>-<과제명>/`.
- **단원설계.md** (선택): **있으면** 수업 운영 회고를 함께 쓴다 — 차시 계획 대비 실제 운영,
  핵심 질문이 실제로 사고를 열었는지, 형성평가→수행평가 리허설 배치가 작동했는지.
- **교사 소감** (선택): 대화로 받은 코멘트. 없으면 데이터 통계만으로 초안을 만들고 소감란은 비워 둔다.

## 절차
1. `산출물/채점표-*.csv`를 읽어 **기계적 통계**를 낸다 (지어내지 않는다):
   - 반별 수준 분포 (요소별 우수/보통/미흡 인원)
   - `[확인 필요]` 빈도 — **어느 요소에서 관찰 근거가 부족했는가** (관찰기록 양식·루브릭 기술의 개선 신호)
   - 합계 분포 (변별이 되었는가 — 상대평가 과목이면 특히)
2. `수행기록/관찰기록-*.csv`에서 기록 밀도(학생당 행 수)를 본다 — 기록이 몰린/빈 구간.
3. **(단원설계.md가 있으면)** 계획 대비 운영을 대조한다 — 판단 근거는 교사 소감이며, 데이터에 없으면 `[확인 필요]`:
   - 차시 계획대로 진행됐는가 — 어긋난 차시와 이유 (시간 부족? 선행 개념 결손?)
   - 핵심 질문이 실제로 사고를 열었는가 (3층위 기준: `references/평가유형-카탈로그.md` §6 — ⚠ 고시 원문 아님)
     — 학생 반응이 남아 있으면 관찰기록에서 **번호로만** 인용 (비식별)
   - 형성평가→수행평가 **리허설 배치**가 작동했는가 — 형성평가에서 드러난 결손이 수행평가 전에 회복됐는가
     (배치 원칙은 참고 — ⚠ 고시 원문 아님)
4. 교사 소감을 받으면 함께 정리한다.
5. `회고.md` 작성 — 구조:
   - **데이터 요약** (위 통계)
   - **잘 작동한 것** / **흔들린 것**
   - **수업 운영 회고** (단원설계.md가 있을 때만) — 계획 대비 어긋난 차시와 이유 / 핵심 질문·형성평가 배치의 작동 여부 / **다음 단원 설계에 반영할 것**
   - **다음에 고칠 것** — 반드시 **대상 파일 명시** (예: `루브릭.json` 배점, `활동지.html` 문항, `해석.md` 개념 구조, `단원설계.md` 차시 배분·핵심 질문, `형성평가-<차시>차시.html` 문항, `references/관찰기록-양식.md`, `CLAUDE.md` 규칙)
   - 하단: "회고 확정 시 다음 학기 준비 단계에서 이 목록을 먼저 읽는다"
6. `node scripts/lint.mjs` 통과 확인.

## 핵심
- 통계는 **채점표 데이터에서만**. 인상 평가는 교사 소감란에만 들어간다.
- 개선 항목은 파일 단위로 — "루브릭을 더 좋게" (✗) / "루브릭.json '몰환산' 보통 기술이 우수와 겹침 → 경계 행동 명시" (○)
- 수업 운영 회고도 같은 규칙 — "질문이 안 먹혔다" (✗) / "단원설계.md 3차시 개념적 질문이 사실 확인으로 소비됨 → 논쟁적 질문으로 교체" (○)
- 회고가 확정되기 전에는 다음 학기 산출물을 복사·재사용하지 않는다.
