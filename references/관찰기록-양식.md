# 관찰기록 양식 — 실시 단계 표준 (진실의 원천)

> 수행평가를 치르는 동안 남기는 **구조화된 관찰 기록**의 표준. `관찰기록` 스킬과 `scripts/lint.mjs`가 이 양식을 근거로 동작한다.
> 원칙: 학기말에 기억으로 복원하지 말고, 그 순간 '쓸 수 있는 형태'로 남긴다.

## 파일 위치·이름

```
assessments/<과목>-<과제명>/수행기록/관찰기록-<반>.csv    (예: 관찰기록-1반.csv)
```

## 열 정의 (4열 고정 — 순서·이름 변경 금지)

| 열 | 형식 | 규칙 |
|---|---|---|
| `번호` | 숫자만 | **비식별 식별자.** 실명·학번·이니셜 금지. lint가 숫자 여부 검사 |
| `날짜` | `M.D` 또는 `YYYY-MM-DD` | 관찰한 그 날. 소급 기록 시에도 실제 관찰일 |
| `관찰한 사실` | 문장 | **행동·발화·산출물 등 사실만.** 해석·평가어("우수함","성실함","뛰어남") 금지 — 평가는 결과 단계에서 루브릭으로 |
| `관련 역량` | 짧은 명사구, `·` 구분 | 예: `데이터해석·의사소통`, `모형화`, `탐구설계` |

## 예시

```csv
번호,날짜,관찰한 사실,관련 역량
12,6.3,적정 종말점 근거를 모둠에 설명,데이터해석·의사소통
7,6.3,"몰 환산 문제에서 단위 소거 과정을 스스로 점검, 오류 발견 후 수정",문제해결·자기점검
21,6.10,반응식 계수를 임의로 맞추다 짝의 지적을 받고 검산 방법을 물어봄,의사소통
```

## 작성 규칙

1. **한 행 = 한 관찰.** 같은 학생이라도 관찰마다 행을 나눈다 (날짜별 근거가 됨).
2. **사실 문장으로.** "발표를 잘함" (✗ 해석) → "발표에서 계수비를 근거로 질량비를 유도해 보임" (○ 사실).
3. **쉼표가 든 문장은 큰따옴표로 감싼다** (CSV 규칙).
4. 모든 학생을 매번 기록할 필요 없다. **눈에 띈 순간을 그때** 적는 것이 목적. 다만 결과 단계에서 기록이 없는 학생·요소는 `[확인 필요]`가 되므로, 채점 전 보완 관찰로 채운다.
5. 부정적 사실도 기록 가능(지도 근거). 단 인격 평가가 아닌 **행동 사실**로.

## 개인정보 (절대 규칙)

- 이 파일을 포함한 **모든 기록에 실명·학번·생년월일 금지.** 번호만 쓴다.
- 번호↔실명 대응표(명렬)는 이 패키지 밖(교무 시스템·교사 수첩)에만 둔다. 저장소·클라우드에 올리지 않는다.
