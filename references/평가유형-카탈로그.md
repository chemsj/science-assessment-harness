# 평가유형 카탈로그 — 준비 단계 '구상'의 근거

> 수행평가로 굴릴 수 있는 **평가유형의 목록**과, 교과서·지도서에서 추출한 요소의 **분류 어휘**를 정의한다.
> `구상` 스킬과 `scripts/lint.mjs`(수행평가-요소 검증)가 이 문서를 근거로 동작한다.
> ※ 이 문서는 고시 원문이 아니라 **패키지 내부 규약**(일반적 평가 원리 정리)이다. 학교 지침과 다르면 학교 지침이 우선.

## 1. 수행평가 유형 10종

| 유형 | 학생 산출물 | 관찰 포인트 (관찰기록 연계) | 유의점 |
|---|---|---|---|
| **실험 수행·보고서** | 실험 보고서, 측정 데이터 | 변인 통제, 측정 정확성, 안전 수칙, 데이터 기록 습관 | 조별 활동 시 개인 기여 관찰 필수 |
| **탐구 설계** | 탐구 계획서(가설·변인·절차) | 가설의 검증 가능성, 변인 설정, 절차의 논리 | 실행 없이 설계만도 평가 가능 |
| **자료 해석** | 해석 답안, 분석 노트 | 표·그래프 독해, 경향 파악, 근거 있는 결론 | 지필과 차별화: 실제 데이터·개방형 |
| **논술·글쓰기** | 과학 글(주장문·설명문) | 개념의 정확성, 근거-주장 연결, 과학 용어 사용 | 서·논술형 의무 비율 반영 가능 |
| **조사·발표** | 발표 자료, 요약문 | 출처의 신뢰성, 내용 구성, 질의응답 대응 | 단회 관찰 위험 → 제출물 병행 |
| **토의·토론** | 논거 메모, 입장문 | 논거의 과학적 타당성, 경청·반론, 합의 과정 | 발화량이 아니라 논거 질을 본다 |
| **모형·표현** | 분자 모형, 인포그래픽, 시각화 | 개념의 정확한 표상, 표현의 창의성 | 미적 완성도가 아니라 개념 표상 중심 |
| **프로젝트·융합** | 프로젝트 보고서, 산출물 | 문제 정의, 과정 관리, 사회·타교과 연결 | 장기 과제 — 중간 관찰 지점 설계 |
| **포트폴리오** | 활동 누적물(활동지·기록 묶음) | 성장 과정, 자기 성찰의 구체성 | 수합 시점·형태를 초기에 고정 |
| **구술·질의응답** | (구술 답변) | 개념 설명력, 즉석 사고, 재구성 능력 | 질문 목록 사전 표준화(공정성) |

## 2. 교과서·지도서 요소의 분류 어휘 (통제 어휘 — lint 검증 대상)

`references/수행평가-요소/<과목>-<출판사>.json`의 `유형` 필드는 아래 8개 값만 쓴다.

| 분류 어휘 | 무엇을 담나 | 주로 이어지는 평가유형 |
|---|---|---|
| `실험` | 실험·탐구 활동 (기구·시약 사용, 측정 포함) | 실험 수행·보고서, 탐구 설계 |
| `자료해석` | 표·그래프·사례 분석 활동 | 자료 해석, 논술·글쓰기 |
| `토의·토론` | 토의·토론·의사결정 활동 | 토의·토론 |
| `조사·발표` | 조사·인터뷰·발표 활동 | 조사·발표, 프로젝트·융합 |
| `글쓰기·읽기` | 읽기자료·과학사·과학과 사회 이슈·글쓰기 소재 | 논술·글쓰기, 구술·질의응답 |
| `모형·표현` | 모형 만들기·그리기·시각화 활동 | 모형·표현 |
| `프로젝트·융합` | 단원 마무리 대형 활동, 창의·융합, 진로 연계 | 프로젝트·융합, 포트폴리오 |
| `평가자료` | **지도서의 평가 관련 자료** (수행평가 예시, 평가 문항, 채점 기준안) | 전 유형 (출판사 제안 그대로 검토) |

## 3. 수행평가-요소 JSON 스키마

```json
{
  "_메타": {
    "과목코드": "화학",
    "출판사": "…",
    "원본": ["교과서 파일명.pdf", "지도서 파일명.pdf"],
    "추출일": "YYYY-MM-DD",
    "비고": "활동명·쪽 수준 목록. 원문 대조 필요. 로컬 전용(공유 저장소 제외)."
  },
  "단원구조": [ { "대단원": "I. …", "영역": "화학의 언어", "쪽": "10-63" } ],
  "요소": [
    {
      "이름": "…(원문 표기 그대로)",
      "유형": "실험",
      "영역": "화학의 언어",
      "출처": "교과서",
      "쪽": 34,
      "개요": "한 문장 요약",
      "수행평가활용": "이 요소로 만들 수 있는 평가 한 줄"
    }
  ]
}
```

- `영역`은 `references/성취기준/<과목>.json`의 영역명만 쓴다 (화학: 화학의 언어 / 물질의 구조와 성질 / 화학 평형 / 역동적인 화학 반응). 애매하면 `[확인 필요]`.
- `쪽`은 반드시 기입 — 교사가 원문과 대조하는 통로다.
- **저작권**: 원본 PDF(`references/_교과서/`)와 이 추출 JSON은 **로컬 전용**. 공유 저장소에 올리지 않는다.

## 방법론 용어 표준 (§4~§6의 근거 이론 — 정식 명칭·약자)

> §4~§6의 설계 방법론은 아래 학계 이론에서 왔다. ⚠ 모두 **고시 원문이 아니다**(방법론 차용). 국내 과학과 적용은 경남교육청 연수자료(임정미·김보경, 2026, `자료분석/`)를 참고했다.
> **표기 규칙**: 처음 나올 때는 정식 명칭을 쓰고 약자를 괄호로 병기한다(예: 개념기반 교육과정과 수업(CBCI)). 이후에는 필요할 때만 약자를 쓴다.

| 정식 명칭 | 약자 | 원저자 | 이 패키지에서 쓰는 곳 |
|---|---|---|---|
| 개념기반 교육과정과 수업 | **CBCI** (Concept-Based Curriculum and Instruction) | Erickson, Lanning & French (2017) | §4 개념적 렌즈, 단원 핵심 아이디어(사실→개념→일반화) |
| 이해 중심 교육과정 (백워드 설계) | **UbD** (Understanding by Design) | Wiggins & McTighe (2005) | 단원설계의 역방향 절차(목표→평가→수업) |
| 수행과제 맥락 틀 | **GRASPS** | Wiggins & McTighe (UbD의 도구) | 평가계획서·단원설계의 수행 맥락 |

## 4. 개념적 렌즈 (개념기반 설계 — 구상의 방향타)

> 근거: 개념기반 교육과정과 수업(CBCI; Erickson, Lanning & French, 2017) + 경남교육청 연수 자료(임정미, 2026, `자료분석/`).
> ⚠ **고시 원문이 아니다.** 렌즈 목록은 설계를 돕는 **참고 도구**이며, 학교·교사가 다른 렌즈를 써도 된다.

**왜 쓰나.** 같은 성취기준도 어떤 **개념적 렌즈**로 보느냐에 따라 학습·평가의 깊이가 달라진다.
사실(성취기준) → **개념(렌즈)** → 일반화(단원 핵심 아이디어)로 끌어올리면, "무엇을 평가할지"가 분명해진다.

**렌즈 후보 10종** (하나를 관점으로 고르고, 1~2개를 연결 개념으로 붙인다):

`시스템` · `균형` · `에너지` · `형태` · `기능` · `구조` · `상호작용` · `모델` · `변화` · `관계`

**두 종류의 '핵심 아이디어'를 구분한다 (환각 방지 핵심):**
| 구분 | 무엇 | 출처 | 근거 라벨 |
|---|---|---|---|
| 교육과정 핵심 아이디어 | 별책9 내용 체계의 과목 단위 원문 문장 | `references/성취기준/<과목>.json`의 `핵심아이디어` | `원문(별책9)` (그대로 인용) |
| 단원 핵심 아이디어 | 교사가 렌즈로 만든 그 단원의 일반화 문장 | 교사 설계(구상 산출물) | `제안` (교사 확정 전 잠정) |

- 교육과정 핵심 아이디어는 **데이터에서 꺼내 쓴다.** 지어내지 않는다.
- 단원 핵심 아이디어는 **교사의 제안**이다 — 교육과정 핵심 아이디어를 참고해 렌즈로 좁힌 문장이며, "이 단원이 끝났을 때 학생이 말할 수 있어야 하는 한 문장"으로 진술한다.
- 진술 조건(Erickson): 두 개 이상 개념 포함 완성 문장 / 시간·장소 초월 보편 적용 / 사실 서술이 아닌 원리.

### 근거 라벨 3종 (모든 산출물 공통 — 환각 방지의 핵심 장치)
> 산출물의 모든 항목은 신뢰 등급이 한눈에 보이도록 아래 3종 중 하나로 출처를 표기한다. 등급 순: **`원문` ＞ `참고` ＞ `제안`**.

| 라벨 | 무엇 | 표기 규칙 |
|---|---|---|
| `원문` | 고시·공식 문서를 **그대로** 인용(성취기준·탐구활동·핵심아이디어·내용체계·기재요령 등) | 변형(요약·각색) 금지. 세부 출처를 괄호로: `원문(별책9)`, `원문(기재요령)` |
| `참고` | 출처가 분명한 **외부 자료**(⚠ 고시 원문 아님) — 교과서·지도서, 연수 자료 | 세부 출처(쪽·저자) 필수: `참고(동아 p.113)`, `참고(임정미)`, `참고(김보경)`. 원본은 로컬 전용(저작권) |
| `제안` | 데이터에 없는 **설계 제안**(단원 핵심 아이디어·렌즈 선택·핵심 질문·차시 배분·매핑 등) | 교사 확정 전 **잠정**임을 명시. 교육과정 원문인 척하지 않는다 |

## 5. 수행지시어–증거 언어 사전 (루브릭 수준 기술의 재료)

> 근거: 경남교육청 연수 자료(임정미, 2026, `자료분석/`). ⚠ **고시 원문이 아니다** — 루브릭 수준 기술을 쓸 때 참고하는 어휘 도구.

**왜 쓰나.** 루브릭 수준을 "잘함/보통"이 아니라 **관찰 가능한 행동(수행지시어)**으로 가른다.
수행지시어에는 위계가 있어, 이 위계를 따라 내려가며 수준을 벌리면 채점자 간 일관성이 올라간다.

**수준을 가르는 지시어 위계** (높은 사고 → 낮은 사고):
`설명한다·논증한다` ＞ `서술한다·적용한다` ＞ `해석한다·분류한다` ＞ `요약한다` ＞ `나열한다·열거한다`

**지시어 사전** (지시어 → 학생 수행에서 포착할 증거 언어):
| 사고 수준 | 지시어 | 학생 수행에서 포착할 증거 |
|---|---|---|
| 기억 | 나열한다 / 열거한다 | 관련 용어·사례를 순서 없이 말한다 / 범주 안 항목을 빠짐없이 든다 |
| 기억 | 기억한다 / 찾는다 / 식별한다 | 정의·공식을 그대로 재생한다 / 자료에서 정답 구절·수치를 찾아낸다 / 보기 중 옳은 것을 구분한다 |
| 이해 | 이해한다 / 요약한다 | 핵심 의미를 자기 말로 진술한다 / 긴 내용을 핵심어로 압축한다 |
| 이해 | 설명한다 / 서술한다 | '왜 그런가'를 근거를 들어 말한다 / 인과관계를 논리적 순서로 풀어낸다 |
| 이해 | 해석한다 / 조직화한다 | 그래프를 문장으로(형태 변환) 바꾼다 / 개념 간 관계를 도식화한다 |
| 적용·분석 | 적용한다 / 예측한다 | 배운 원리를 낯선 맥락에 써서 결과를 내다본다 |
| 평가·창조 | 검증한다 / 논증한다 | 모델에서 예측을 스스로 도출한다 / 데이터로 일치·불일치 이유를 설명한다 / 모델의 한계·조건을 인식한다 |

**쓰는 법.** 한 평가요소의 최고 수준을 상위 지시어(설명·논증)로 쓰고, 아래 수준으로 내려가며 지시어를 낮춰 기술한다.
예: 우수 "…근거를 들어 **설명한다**" → 보통 "…핵심 요소 중심으로 **요약한다**" → 미흡 "…단어를 **나열한다**".

## 6. 핵심 질문 3층위 (수업 설계 — 질문 설계의 재료)

> 근거: 개념기반 교육과정과 수업(CBCI; Erickson, Lanning & French, 2017)의 질문 유형 + 경남교육청 연수 자료(김보경 단원 설계안의 핵심 질문 사례, `자료분석/`).
> ⚠ **고시 원문이 아니다** — 차시 질문을 설계할 때 참고하는 패키지 내부 규약이다.
> ⚠ 김보경 설계안은 **2015 개정 기반** — 질문 배치의 방법론만 차용하고, 그 안의 성취기준 코드·단원은 전용하지 않는다.

**왜 쓰나.** 단원 핵심 아이디어(일반화)에 학생을 도달시키는 사다리가 **질문**이다.
사실 질문만 있으면 수업이 암기에 머물고, 논쟁 질문만 있으면 근거 없는 공중전이 된다 —
세 층위를 **섞어 배치**해 사실 → 개념 → 입장의 순서로 사고를 끌어올린다.

**3층위** (화학 평형 단원 예시):
| 층위 | 정의 | 예시 | 어울리는 평가유형 |
|---|---|---|---|
| **사실적 질문** | 정답이 있는 질문. 지식·이해 요소의 확인 — 개념 질문의 발판 | "평형 상수는 어떤 자료로 구하는가?" | 자료 해석, 구술·질의응답 |
| **개념적 질문** | 개념 간 관계를 묻는 질문. 시간·소재를 초월해 **전이**되며, 단원 핵심 아이디어로 수렴 | "계는 왜 변화를 완화하는 방향으로 움직이는가?" | 탐구 설계, 모형·표현, 실험 수행·보고서 |

**쓰는 법.** `단원설계` 스킬이 차시표의 핵심 질문을 이 3층위로 배치하고, `피드백` 스킬이 층위를 한 단계 올리는 질문을 만들 때 참고한다.
- 사실적 질문이 확인하는 지식·이해 요소는 `references/성취기준/<과목>.json`의 `내용체계`(별책9 원문)에서 **꺼내 쓴다** — 데이터에 없는 요소는 `[확인 필요]`.
- 도입 차시는 사실적 질문으로 발판을 놓고, 중반에 개념적 질문, 종반(수행·토론 차시)에 논쟁적 질문을 둔다 — 순서는 교사가 조정한다.
- **개념적 질문이 단원 핵심 아이디어(§4)의 절(節)과 대응하는지 확인한다.** 어느 절에도 닿지 않는 질문은 배치 이유를 다시 본다.
- 질문 문장과 층위 배치는 전부 근거 라벨 `제안`(교사 설계 판단, 확정 전 잠정)이다 — 교육과정 원문에 있는 분류인 척하지 않는다.
