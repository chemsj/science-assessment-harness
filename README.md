# 과학과 수행평가 작업틀 (평가 Harness) — 공유판

> 2022 개정 교육과정 **과학과** 수행평가를 "매 학기 맨땅에서"가 아니라 **한 번 만든 틀에 걸어서** 반복하는 Claude Code 작업 환경입니다.
> 2026. 7. 4. 「저경력 과학교사 DX 역량 강화 직무연수」 배포 자료 (경해여자고등학교 최수진).

## 가져오기 — 3스텝 (연수 슬라이드 10장)

**방법 A — 앱에서 말로 (권장)**
1. Claude **데스크톱 앱**의 **Code 탭**에서 새 작업 폴더를 만들어 선택합니다 (예: `문서\평가작업틀`).
2. 이렇게 말합니다: **"이 저장소를 통째로 가져와서 이 폴더에 구성해줘: `github.com/chemsj/science-assessment-harness`"** — 폴더 구성까지 전부 알아서 됩니다.
3. 우리 반에 맞게 **한 곳만 수정**합니다 — `CLAUDE.md`의 학교·학년·반영 비율. (이것도 말로: "CLAUDE.md를 우리 학교 기준으로 고쳐줘 — ○○고, 1학년, 수행 40%")

**방법 B — ZIP으로 (git 없이)**
1. 이 페이지 상단 초록색 **`<> Code` 버튼 → `Download ZIP`** 클릭 → 압축 해제.
2. 풀린 폴더를 원하는 위치로 옮기고, Claude 앱 **Code 탭**에서 그 폴더를 선택하면 끝.
3. 이후는 방법 A의 3번과 동일.

## 내 교과서 연결하기 (선택 — 저작권상 각자)

이 저장소에는 교과서 관련 자료가 **일부러 빠져** 있습니다. 소속교 교과서로 직접 채우면 '구상(아이디어)' 단계가 훨씬 풍부해집니다.

1. `references/_교과서/` 폴더에 **소속교 교과서·지도서 PDF**를 복사해 넣습니다 (파일 탐색기로 끌어다 놓기).
2. 앱에서 말합니다: **"references/_교과서/의 교과서에서 수행평가 요소를 추출해서 `references/수행평가-요소/<과목>-<출판사>.json`으로 만들어줘"** (활동명·유형·영역·쪽이 정리됩니다)
3. ⚠ **저작권 주의**: 이 두 폴더는 **내 컴퓨터 전용**입니다 — 동료 공유·클라우드 공개·저장소 업로드 금지. (.gitignore가 실수 업로드를 한 번 더 막아줍니다)

이후는 자연어로 부릅니다 (예): "화학 화학 평형 단원 **평가계획서** 만들어줘" → "**루브릭**도" → "이 수행 메모를 1반 **관찰기록**으로" → "1반 기록을 루브릭에 맞춰 **채점표**로".
사용법 전체는 [PROJECT.md](PROJECT.md), 전체 그림은 `패키지-이해.html`(브라우저로 열기)를 보세요.

- 고시·지침 원문 PDF(`references/_원문/`) — [NCIC 국가교육과정정보센터](https://ncic.re.kr) · [학교생활기록부 종합지원포털](https://star.moe.go.kr) · 소속 교육청 누리집에서 내려받으세요

## 원칙 (반드시)

1. **모든 생성물은 초안** — 확정은 교사(편집장)가 합니다.
2. **비식별** — 학생은 번호로만. 실명·학번을 어떤 파일에도 넣지 않습니다. 수업 녹음·전사 파일은 클라우드 동기화 폴더 **밖**에 둡니다.
3. **사실은 데이터에서** — 성취기준·평가기준은 `references/`에서 꺼내 쓰고, 없으면 `[확인 필요]`로 비웁니다.
4. 산출물을 만들면 검증기 **통과**를 확인합니다 — 터미널 없이 **"검증(lint) 돌려서 확인해줘"** 라고 말하면 됩니다. (CLI 사용자는 `node scripts/lint.mjs`)
5. 반영비율·시기 등 행정 수치는 **소속교 학업성적관리규정이 최종**입니다.
